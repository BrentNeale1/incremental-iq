  <action>
    **Extend `apps/web/lib/store/dashboard.ts`:**
    Add to DashboardState interface (per RESEARCH.md Pattern 9):
    - `selectedMarket: string | null` — null means "All Markets"
    - `markets: Array<{ id: string; countryCode: string; displayName: string; campaignCount: number }>` — loaded from API
    - `outcomeMode: 'ecommerce' | 'lead_gen'` — loaded from tenant preferences
    - `setSelectedMarket: (marketId: string | null) => void`
    - `setMarkets: (markets: DashboardState['markets']) => void`
    - `setOutcomeMode: (mode: 'ecommerce' | 'lead_gen') => void`

    Add `selectedMarket` to the persist partialize config so market selection persists across navigation (per user decision).

    **Create `apps/web/hooks/useMarkets.ts`:**
    TanStack Query hook wrapping GET /api/markets?tenantId=...
    - On success, call setMarkets() on the dashboard store
    - Returns { markets, isLoading }
    - staleTime: 5 minutes

    **Create `apps/web/hooks/useOutcomeMode.ts`:**
    Hook that provides:
    - `outcomeMode` from dashboard store
    - `terms`: object mapping outcome mode to display strings: { revenue: 'Revenue' | 'Leads', incrementalRevenue: 'Incremental Revenue' | 'Incremental Leads', conversion: 'Sale' | 'Lead' }
    - Loaded from /api/tenant/preferences on mount, updates store

    **Create `apps/web/app/api/tenant/preferences/route.ts` — GET + PUT:**
    - GET: return tenant outcomeMode from tenants table
    - PUT: update outcomeMode on tenants table, return updated value

    **Extend ALL dashboard API routes with optional `?marketId=` filter:**
    For each of these routes: kpis, campaigns, incrementality, seasonality, saturation, recommendations:
    1. Read `marketId` from searchParams (optional, null = all markets)
    2. When marketId is provided: add INNER JOIN on campaign_markets to filter campaign_metrics by market (RESEARCH.md Pattern 7)
    3. When marketId is null: no filter (all markets combined)
    4. For incrementality route: also filter incrementality_scores by marketId column
    5. For saturation route: filter via campaign_markets join
    6. For recommendations route: filter underlying score/saturation queries by market

    The JOIN pattern for market filtering (add to each route where applicable):
    ```typescript
    if (marketId) {
      query.innerJoin(
        campaignMarkets,
        and(
          eq(campaignMarkets.campaignId, campaignMetrics.campaignId),
          eq(campaignMarkets.marketId, marketId),
        )
      );
    }
    ```

    Use explicit TypeScript types on all withTenant() calls.
  </action>

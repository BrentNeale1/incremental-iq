    Install GA4 packages: `pnpm add @google-analytics/admin @google-analytics/data` in packages/ingestion.

    **Create `packages/ingestion/src/connectors/ga4.ts` — GA4Connector class:**

    Uses `@google-analytics/admin` v9 and `@google-analytics/data` v5. Follow RESEARCH.md Patterns 2 and 3.

    Methods:
    1. `listProperties(accessToken: string)` — Call `analyticsAdminClient.listProperties()` with no parent filter to get all GA4 properties the user has access to. Return `Array<{ propertyId: string, displayName: string }>`. Per RESEARCH.md Pitfall 5: if only one property, auto-select.

    2. `listKeyEvents(accessToken: string, propertyId: string)` — Call `adminClient.listKeyEventsAsync({ parent: 'properties/{propertyId}' })`. Return `Array<{ eventName: string, countingMethod: string }>`. Use `keyEvents` API (NOT deprecated `conversionEvents` — RESEARCH.md State of the Art).

    3. `fetchLeadCounts(accessToken: string, propertyId: string, selectedEventNames: string[], dateRange: { start: string, end: string })` — Call `dataClient.runReport()` with dimensions `['date', 'eventName']`, metric `['eventCount']`, dimensionFilter using `inListFilter` with all selected events in a single request (RESEARCH.md Pitfall 2 — quota management). Sum event counts across all selected events per date. CRITICAL: convert GA4 date format from 'YYYYMMDD' to 'YYYY-MM-DD' (RESEARCH.md Pitfall 1). Return `Map<string, number>` (date -> total leads).

    4. `refreshTokenIfNeeded(accessToken: string, refreshToken: string)` — Standard Google OAuth token refresh via `https://oauth2.googleapis.com/token`. Return new accessToken if refreshed, original if not expired.

    Auth client initialization: Use `google-auth-library` OAuth2Client with user's access token + refresh token. Same pattern as Google Ads but with different scope. Do NOT use developer token (GA4 does not require one — RESEARCH.md Pattern 1).

    **Create `packages/ingestion/src/normalizers/ga4.ts` — processGA4Sync():**

    Follow the two-stage pipeline pattern from normalizers/meta.ts:
    1. Load integration record, decrypt token
    2. Call refreshTokenIfNeeded, update integration if refreshed
    3. Read selectedEventNames from integration.metadata.selectedEventNames (string[])
    4. Read propertyId from integration.metadata.propertyId (string)
    5. Call fetchLeadCounts for date range
    6. Stage 1: Store raw GA4 response in raw_api_pulls (source='ga4', apiVersion='data-v1beta')
    7. Stage 2: For each date, upsert to campaign_metrics:
       - For lead-gen tenants, store lead count in `directConversions` column (not directRevenue) per RESEARCH.md Pattern 10
       - Use a synthetic campaign row for GA4 leads (similar to Shopify 'shopify-revenue' pattern from Phase 2) — one 'ga4-leads' campaign per tenant
       - 4-column upsert conflict target (tenantId, campaignId, date, source='ga4')
    8. All DB operations via withTenant()

    **Update connectors/index.ts:**
    Add `case 'ga4': return new GA4Connector()` to getConnector registry.

    Wrap all API calls with p-retry (same pattern as meta.ts connector).

  <action>
    **Create `apps/web/app/api/oauth/ga4/route.ts` — OAuth initiation:**
    Follow exact pattern from `apps/web/app/api/oauth/google/route.ts` but with GA4-specific scope.
    - Scope: `https://www.googleapis.com/auth/analytics.readonly` (NOT analytics.edit — RESEARCH.md State of the Art)
    - Use env vars: GA4_CLIENT_ID (or fall back to GOOGLE_ADS_CLIENT_ID if same Google Cloud project), GA4_CLIENT_SECRET
    - access_type: 'offline', prompt: 'consent' (required for refresh_token)
    - State parameter via generateState(tenantId) from oauth-helpers.ts
    - redirect_uri: `${NEXT_PUBLIC_APP_URL}/api/oauth/ga4/callback`
    - GA4 OAuth is SEPARATE from Google Ads OAuth (different scope, possibly different Google account — RESEARCH.md Pattern 1)

    **Create `apps/web/app/api/oauth/ga4/callback/route.ts` — OAuth callback:**
    Follow pattern from `apps/web/app/api/oauth/google/callback/route.ts`:
    1. Verify state parameter via verifyState()
    2. Exchange authorization code for tokens via `https://oauth2.googleapis.com/token`
    3. Call saveIntegration() with platform='ga4', store accessToken + refreshToken encrypted
    4. Store empty selectedEventNames in metadata initially — user selects events in next step
    5. Do NOT trigger backfill from callback (same anti-pattern rule as Phase 2)
    6. Return JSON with integrationId and redirect to property selection step

    **Create `apps/web/app/api/ga4/properties/route.ts` — GET:**
    1. Accept `?integrationId=` query param
    2. Load integration, decrypt token
    3. Call GA4Connector.listProperties(accessToken)
    4. Return JSON array of { propertyId, displayName }
    5. If only one property, include `autoSelected: true` flag in response

    **Create `apps/web/app/api/ga4/events/route.ts` — GET:**
    1. Accept `?integrationId=&propertyId=` query params
    2. Load integration, decrypt token
    3. Call GA4Connector.listKeyEvents(accessToken, propertyId)
    4. Return JSON array of { eventName, countingMethod }
    5. Also accept POST to save selected events: update integration.metadata.selectedEventNames and integration.metadata.propertyId

    All routes use withTenant() for RLS where accessing DB.
  </action>

- Return dict with same shape as compute_incrementality: lift_mean, lift_lower, lift_upper, confidence, cumulative_lift, pre_period_mean, post_period_mean, counterfactual_mean (= pre_period_mean for raw), diagnostics={'method': 'rolling_mean_comparison'}

**GREEN — Implement (packages/analysis/models/hierarchical.py):**

`hierarchical_pooled_estimate(campaigns: list[dict], cluster_key: str) -> list[dict]`
- Each campaign dict contains: campaign_id, metrics (list[MetricRow]), intervention_date, data_points_count
- For campaigns with >= 30 data points, compute individual ITS estimates via `compute_incrementality()`
- Build a PyMC hierarchical model:
  - Hyperpriors: `mu_cluster = pm.Normal('mu_cluster', mu=0, sigma=1)` and `sigma_cluster = pm.HalfNormal('sigma_cluster', sigma=0.5)`
  - Per-campaign: `lift_i = pm.Normal(f'lift_{i}', mu=mu_cluster, sigma=sigma_cluster, observed=individual_lift_mean)` for data-rich campaigns
  - For sparse campaigns (< 30 points): `lift_i = pm.Normal(f'lift_{i}', mu=mu_cluster, sigma=sigma_cluster)` — no observation, so posterior is pulled toward cluster mean
  - Sample with `pm.sample(draws=500, tune=200, return_inferencedata=True)`
- Extract posterior for each campaign: lift_mean = posterior mean, lift_lower/lift_upper = HDI
- For sparse campaigns: status = 'pooled_estimate', confidence is derived from posterior width (narrower = higher confidence but always lower than individually-estimated campaigns)
- Return list of dicts: one per campaign with campaign_id, lift_mean, lift_lower, lift_upper, confidence, status ('scored' for data-rich, 'pooled_estimate' for sparse)

Per user decision: "marketers always get a directional signal, never a dead-end 'insufficient data' wall" — the pooled_estimate gives sparse campaigns a cluster-informed directional signal with honestly wide credible intervals.

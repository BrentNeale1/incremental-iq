**packages/ingestion/src/scoring/budget-detection.ts:**

`detectBudgetChanges(db: DrizzleDB, tenantId: string, campaignId: string, thresholdPct: number = 0.25): Promise<BudgetChangeEvent | null>`

Implementation:
1. Query campaign_metrics for the last 60 days of daily spend for the campaign, ordered by date
2. Apply 3-day rolling average smoothing to spend data BEFORE threshold comparison (Pitfall 5: avoids billing cycle false positives). Smoothing can be done in-query with a window function: `AVG(spend_usd) OVER (ORDER BY date ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING)`
3. Compare 14-day rolling averages: avg of days 28-15 ago (pre) vs avg of days 14-1 ago (post) from the smoothed series
4. If |post - pre| / pre > thresholdPct, return a BudgetChangeEvent with: changeDate (approximate midpoint), spendBeforeAvg, spendAfterAvg, changePct (signed: positive = increase, negative = decrease)
5. If no significant change detected, return null

`persistBudgetChange(db: DrizzleDB, event: BudgetChangeEvent): Promise<void>`
- Insert into budget_changes table with source='auto_detected', status='pending_analysis'
- Use Drizzle insert into budgetChanges schema

`scanAllCampaignsForBudgetChanges(db: DrizzleDB, tenantId: string): Promise<BudgetChangeEvent[]>`
- Query all active campaigns for the tenant (campaigns with spend in last 30 days)
- Call detectBudgetChanges for each
- Filter out nulls, persist each detected change
- Return list of detected changes

}

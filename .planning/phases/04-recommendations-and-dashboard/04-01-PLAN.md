**A. Create ThemeProvider (apps/web/components/layout/ThemeProvider.tsx):**
- 'use client' directive
- Wraps next-themes ThemeProvider with attribute="class", defaultTheme="light", enableSystem, disableTransitionOnChange
- Export as named `ThemeProvider`

**B. Create QueryProvider (apps/web/components/layout/QueryProvider.tsx):**
- 'use client' directive
- Uses getQueryClient() singleton pattern from lib/query/client.ts
- Wraps children in QueryClientProvider
- Includes ReactQueryDevtools in development

**C. Create QueryClient singleton (apps/web/lib/query/client.ts):**
- SSR-safe singleton: new instance per server request, cached instance in browser
- Default staleTime: 60_000 (1 minute)
- Pattern from research: `typeof window === 'undefined'` check

**D. Create Zustand dashboard store (apps/web/lib/store/dashboard.ts):**
- `useDashboardStore` with:
  - `dateRange: { from: Date; to: Date }` — default last 30 days (not persisted)
  - `comparisonRange: { from: Date; to: Date } | null` — null by default
  - `comparisonEnabled: boolean` — false by default
  - `viewMode: 'executive' | 'analyst'` — persisted, default 'executive'
  - `kpiOrder: string[]` — persisted, default ['spend', 'revenue', 'roas', 'incremental_revenue']
  - Setters for each field
- Use persist middleware with `partialize` to persist only viewMode and kpiOrder
- CRITICAL: Use `skipHydration: true` to avoid Next.js SSR hydration mismatch (Pitfall 2 from RESEARCH.md)

**E. Create Zustand notifications store (apps/web/lib/store/notifications.ts):**
- `useNotificationStore` with:
  - `unreadCount: number`
  - `setUnreadCount: (count: number) => void`
- Not persisted — fetched from API on mount

**F. Wire providers into root layout (apps/web/app/layout.tsx):**
- Import and wrap: ThemeProvider > QueryProvider > {children}
- Keep Inter + Manrope font variables on html className
- Import globals.css

**G. Create DB schema — notifications table (packages/db/src/schema/notifications.ts):**
- `notifications` pgTable with: id (uuid pk), tenantId (uuid not null), type (text: 'anomaly' | 'recommendation' | 'seasonal' | 'data_health'), message (text), linkPath (text nullable), read (boolean default false), createdAt (timestamp with tz, defaultNow)
- Index on (tenantId, read, createdAt) for unread notification queries
- RLS policy matching existing pattern: pgPolicy restrictive, using appRole, for select/insert/update where tenantId = current_setting('app.tenant_id')::uuid

**H. Create DB schema — user_preferences table (packages/db/src/schema/user-preferences.ts):**
- `userPreferences` pgTable with: tenantId (uuid pk), kpiOrder (jsonb string[] default), viewMode (text default 'executive'), theme (text default 'light'), notificationPrefs (jsonb with typed structure), brandColors (jsonb nullable), updatedAt (timestamp with tz, defaultNow)
- RLS policy: same pattern as notifications

**I. Update packages/db/src/schema/index.ts:**
- Add re-exports for notifications and userPreferences

**J. Create migration (packages/db/migrations/0004_phase4_dashboard.sql):**

- CREATE INDEX notifications_tenant_unread_idx ON notifications (tenant_id, read, created_at)
- Follow exact pattern from 0003_statistical_engine.sql

  <action>
Create all six schema definition files. Every table that stores tenant data MUST include a `pgPolicy('tenant_isolation', ...)` with `as: 'restrictive'`, `for: 'all'`, `to: appRole`, `using: sql\`tenant_id = current_setting('app.current_tenant_id')::uuid\``, `withCheck: sql\`tenant_id = current_setting('app.current_tenant_id')::uuid\``. Import `appRole` from `./roles` in each file.

1. **tenants.ts** — `tenants` table. Fields: id (uuid pk defaultRandom), name (text notNull), slug (text notNull unique), plan (text notNull default 'starter'), analysisUnlocked (boolean default false notNull), analysisUnlockedAt (timestamp with timezone, nullable), createdAt/updatedAt (timestamp with timezone, defaultNow, notNull). NO RLS on this table -- it is the root of the isolation hierarchy. (See RESEARCH.md "Tenants / Organizations Table").

2. **campaigns.ts** — Three tables in this file:
   - `campaigns`: id (uuid pk), tenantId (uuid notNull), name (text notNull), source (text notNull -- 'google_ads' | 'meta' | etc), externalId (text notNull), status (text), createdAt (timestamp tz). RLS policy.
   - `adSets`: id (uuid pk), tenantId (uuid notNull), campaignId (uuid, references campaigns.id), externalId (text notNull), name (text), status (text), createdAt (timestamp tz). RLS policy.
   - `ads`: id (uuid pk), tenantId (uuid notNull), adSetId (uuid, references adSets.id), creativeId (uuid, references creatives.id -- ARCH-01 link), externalId (text notNull), name (text), status (text), createdAt (timestamp tz). RLS policy.

3. **creatives.ts** (ARCH-01) — `creatives` table with all metadata fields from RESEARCH.md Pattern 4: id, tenantId, externalId, source, name, format, headline, primaryText, description, callToAction, imageUrl, videoUrl, thumbnailUrl, aspectRatio, durationSeconds (numeric 6,1), externalMetadata (jsonb), createdAt, updatedAt. RLS policy.

4. **metrics.ts** (ARCH-02) — `campaignMetrics` table with dual attribution columns from RESEARCH.md Pattern 2+3: date (date notNull -- required for hypertable), tenantId (uuid notNull), campaignId (uuid notNull), source (text notNull), spendUsd (numeric 12,4). Direct columns: directRevenue (14,4), directConversions (10,2), directRoas (8,4). Modeled columns (all nullable, populated by Phase 3): modeledRevenue, modeledConversions, modeledRoas, modeledIncrementalLift (8,6), modeledLiftLower, modeledLiftUpper, modeledConfidence (5,4), modeledAt (timestamp tz). Impression data: impressions (14,0), clicks (12,0), ctr (8,6), cpm (10,4). Add `uniqueIndex('campaign_metrics_unique').on(t.tenantId, t.campaignId, t.date, t.source)` (Pitfall 5 from RESEARCH.md). RLS policy. Do NOT add a UUID primary key -- hypertable pattern uses the unique index as dedup constraint (Open Question 1 from RESEARCH.md).

5. **raw-pulls.ts** — `rawApiPulls` table from RESEARCH.md Pattern 6: id (uuid pk), tenantId, source, pulledAt (timestamp tz notNull -- hypertable time column), apiVersion, attributionWindow (Pitfall 3 -- store this!), apiParams (jsonb notNull), payload (jsonb notNull), normalized (boolean default false notNull), normalizedAt, schemaVersion. RLS policy.

6. **ingestion-coverage.ts** (ARCH-03) — `ingestionCoverage` table from RESEARCH.md Pattern 5: id (uuid pk), tenantId, source, coverageDate (date notNull), status (text notNull -- 'complete' | 'partial' | 'failed' | 'pending'), recordCount (numeric 12,0), ingestedAt (timestamp tz), notes (text). RLS policy.

After creating all files, update `packages/db/src/schema/index.ts` to re-export everything.

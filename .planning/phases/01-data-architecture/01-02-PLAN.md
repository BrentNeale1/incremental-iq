This prevents the table owner from bypassing RLS (Pitfall 1 from RESEARCH.md). Add these ALTER statements to the end of the generated migration file OR create a separate custom migration.

**Step 3: Create custom TimescaleDB hypertable migration.**

Run `pnpm drizzle-kit generate --custom --name=hypertables` to create an empty custom migration file. Populate it with the following SQL (from RESEARCH.md Pattern 2):

```sql
-- Enable TimescaleDB extension
CREATE EXTENSION IF NOT EXISTS timescaledb;

-- Convert campaign_metrics to hypertable
-- chunk_time_interval = 1 month (appropriate for MVP scale ~1-10K rows/day)
SELECT create_hypertable(
  'campaign_metrics',
  'date',
  chunk_time_interval => INTERVAL '1 month',
  if_not_exists => TRUE
);

-- Compression: compress chunks older than 90 days
ALTER TABLE campaign_metrics SET (
  timescaledb.compress,
  timescaledb.compress_segmentby = 'tenant_id, campaign_id',
  timescaledb.compress_orderby = 'date DESC'
);
SELECT add_compression_policy('campaign_metrics', INTERVAL '90 days');

-- Convert raw_api_pulls to hypertable (append-only, benefits from time partitioning)
SELECT create_hypertable(
  'raw_api_pulls',
  'pulled_at',
  chunk_time_interval => INTERVAL '1 week',
  if_not_exists => TRUE
);
```

NOTE: Do NOT include the continuous aggregate from the research template yet. Continuous aggregates reference modeled columns that will be NULL until Phase 3. The aggregate can be added when the statistical engine populates those columns.

Create the migration runner script at packages/db/src/migrate.ts:

```typescript
import { drizzle } from 'drizzle-orm/postgres-js';
import { migrate } from 'drizzle-orm/postgres-js/migrator';
import postgres from 'postgres';

// Non-pooling, single connection for migrations (sequential execution)
const migrationClient = postgres(process.env.DATABASE_URL!, { max: 1 });

async function runMigrations() {
  const db = drizzle(migrationClient);
  console.log('Running migrations...');
  await migrate(db, { migrationsFolder: './migrations' });
  await migrationClient.end();
  console.log('Migrations complete');
}

runMigrations().catch((err) => {
  console.error('Migration failed:', err);
  process.exit(1);
});
```

(See RESEARCH.md "Migration Execution Script" for this exact pattern.)

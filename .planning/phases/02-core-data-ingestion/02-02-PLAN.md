Purpose: OAuth callbacks MUST be HTTP endpoints — they cannot be background workers. The ad platforms redirect users to these callback URLs after authorization. **apps/web/lib/oauth-helpers.ts:**
Shared helpers used by all three callback routes:
- `saveIntegration(params: { tenantId, platform, accountId, accountName, accessToken, refreshToken, tokenExpiresAt, metadata })` — encrypts tokens using `encryptToken` from `@incremental-iq/ingestion`, inserts into integrations table using Drizzle, returns the integration record. Uses `withTenant` for RLS context.
- `generateState(tenantId: string): string` — creates a signed state parameter (HMAC-SHA256) encoding the tenantId for OAuth CSRF protection. Uses `process.env.OAUTH_STATE_SECRET`.
- `verifyState(state: string): { tenantId: string }` — verifies and decodes the state parameter.

**OAuth Initiation Routes (GET handlers):**

Each initiation route (`/api/oauth/{platform}/route.ts`) exports a GET handler that:
1. Reads `tenantId` from query params (required — 400 if missing)
2. Generates a signed state parameter using `generateState(tenantId)`
3. Redirects to the platform's OAuth authorization URL with appropriate scopes

- **Meta** (`/api/oauth/meta`): Redirect to `https://www.facebook.com/v23.0/dialog/oauth` with `client_id=FACEBOOK_APP_ID`, `redirect_uri`, `state`, `scope=ads_read,ads_management`. Read `FACEBOOK_APP_ID` and `FACEBOOK_APP_SECRET` from env.
- **Google** (`/api/oauth/google`): Redirect to `https://accounts.google.com/o/oauth2/v2/auth` with `client_id=GOOGLE_ADS_CLIENT_ID`, `redirect_uri`, `state`, `scope=https://www.googleapis.com/auth/adwords`, `access_type=offline`, `prompt=consent`. Read `GOOGLE_ADS_CLIENT_ID` and `GOOGLE_ADS_CLIENT_SECRET` from env.
- **Shopify** (`/api/oauth/shopify`): Read `shop` from query params (required). Redirect to `https://{shop}/admin/oauth/authorize` with `client_id=SHOPIFY_API_KEY`, `scopes=read_orders,read_all_orders`, `redirect_uri`, `state`. Read `SHOPIFY_API_KEY` and `SHOPIFY_API_SECRET` from env.

**OAuth Callback Routes (GET handlers):**

Each callback route (`/api/oauth/{platform}/callback/route.ts`) exports a GET handler that:
1. Reads `code` and `state` from query params
2. Verifies state parameter using `verifyState(state)` — 403 if invalid
3. Exchanges the authorization code for tokens via POST to the platform's token endpoint
4. Calls `saveIntegration(...)` to encrypt and store credentials
5. CRITICAL: Does NOT initiate backfill from the HTTP handler (per RESEARCH.md anti-pattern). Instead, returns a redirect to a success page or JSON response indicating the integration was connected. Backfill is triggered by the scheduler (Plan 06).

Platform-specific token exchange:
- **Meta callback**: POST to `https://graph.facebook.com/v23.0/oauth/access_token` with `client_id`, `client_secret`, `redirect_uri`, `code`. Then exchange short-lived token for long-lived token via `https://graph.facebook.com/v23.0/oauth/access_token?grant_type=fb_exchange_token&client_id=...&client_secret=...&fb_exchange_token=...`. Store the long-lived token (60-day expiry). Also fetch the ad account ID using `https://graph.facebook.com/v23.0/me/adaccounts` and store in metadata.
- **Google callback**: POST to `https://oauth2.googleapis.com/token` with `client_id`, `client_secret`, `redirect_uri`, `code`, `grant_type=authorization_code`. Store both access_token and refresh_token. Also call `customer.listAccessibleCustomers()` pattern to get customer IDs and store in metadata (per RESEARCH.md Pitfall 5 — loginCustomerId for MCC accounts).
- **Shopify callback**: POST to `https://{shop}/admin/oauth/access_token` with `client_id`, `client_secret`, `code`. Store access_token and (if present) refresh_token. Store shop domain in metadata.

All env vars needed:
- `FACEBOOK_APP_ID`, `FACEBOOK_APP_SECRET`
- `GOOGLE_ADS_CLIENT_ID`, `GOOGLE_ADS_CLIENT_SECRET`, `GOOGLE_ADS_DEVELOPER_TOKEN`
- `SHOPIFY_API_KEY`, `SHOPIFY_API_SECRET`
- `TOKEN_ENCRYPTION_KEY` (32 bytes hex)
- `OAUTH_STATE_SECRET` (for HMAC state parameter)
- `NEXT_PUBLIC_APP_URL` (for constructing redirect_uri)

After creating all files, run `pnpm install` from repo root to link workspace packages.

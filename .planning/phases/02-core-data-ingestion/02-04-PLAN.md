**packages/ingestion/src/connectors/google-ads.ts:**

Implement `GoogleAdsConnector` implementing the PlatformConnector interface. Use `google-ads-api` (Opteo) v23.

**Constructor / initialization:**
- Create `GoogleAdsApi` instance with `client_id` (GOOGLE_ADS_CLIENT_ID), `client_secret` (GOOGLE_ADS_CLIENT_SECRET), `developer_token` (GOOGLE_ADS_DEVELOPER_TOKEN) from process.env
- Create `Customer` instance using `customerId` from config and `loginCustomerId` from config.credentials.metadata (for MCC accounts per RESEARCH.md Pitfall 5)
- Use `refresh_token` from decrypted credentials

**fetchCampaigns(config):**
1. Query using GAQL:
```
SELECT campaign.id, campaign.name, campaign.status
FROM campaign
WHERE campaign.status != 'REMOVED'
```
2. Return campaign data mapped to RawCampaignData
3. Google Ads does not have the same ad set/ad hierarchy granularity as Meta for initial ingestion — fetch campaign-level data only (ad groups can be added later if needed)

**fetchMetrics(config, dateRange):**
1. Use `customer.report()` with GAQL:
```
SELECT
  campaign.id, campaign.name,
  metrics.cost_micros, metrics.clicks, metrics.impressions,
  metrics.ctr, metrics.average_cpm,
  segments.date
FROM campaign
WHERE segments.date BETWEEN '{start}' AND '{end}'
  AND campaign.status != 'REMOVED'
```
2. Google Ads API has no practical historical lookback limit — no need for async reporting
3. For very large date ranges (>1 year), chunk into quarterly windows to avoid response size issues. Use `date-fns` `eachQuarterOfInterval` for chunking.
4. Apply `p-retry` with 5 retries, 10s base timeout, randomize jitter
5. Return RawMetricData array with daily rows

**CRITICAL: cost_micros conversion:**
- Google Ads reports costs in micros (1,000,000 micros = $1 USD)
- The connector should return raw micros; the normalizer handles conversion

**refreshTokenIfNeeded(config):**
- Google OAuth refresh tokens do not expire (unless revoked). The `google-ads-api` library handles access token refresh automatically using the refresh_token.
- Return credentials unchanged. If an API call fails with AUTH error, throw to trigger re-authorization flow.

**Update packages/ingestion/src/connectors/index.ts:**
Register `GoogleAdsConnector` in the connector registry for platform `'google_ads'`.

**packages/ingestion/src/normalizers/google-ads.ts:**

Follow the same two-stage pipeline pattern as normalizers/meta.ts:

**Stage 1 — `storeRawPull(params)`:** Same pattern as Meta — insert raw API response into rawApiPulls with `source: 'google_ads'`, `apiVersion: 'v23'`, `normalized: false`. Use withTenant for RLS.

**Stage 2 — `normalizeGoogleAdsMetrics(params: { tenantId, rawPullId, payload }): Promise<number>`:**
1. Parse the GAQL report results
2. For each row, map to NormalizedMetric:
   - `date` from `segments.date`
   - `campaignId` — look up internal UUID from campaigns table by matching `externalId` to Google's `campaign.id` and `source = 'google_ads'`
   - `source` = `'google_ads'`
   - `spendUsd` = `metrics.cost_micros / 1_000_000` — CRITICAL conversion from micros to dollars
   - `impressions` from `metrics.impressions`
   - `clicks` from `metrics.clicks`
   - `ctr` from `metrics.ctr` (Google provides this as a decimal, not percentage)
   - `cpm` from `metrics.average_cpm / 1_000_000` (also in micros)
   - `directRevenue`, `directConversions`, `directRoas` = NULL (ad platform, not revenue source)
3. Upsert into `campaignMetrics` using `onConflictDoUpdate` with target `[tenantId, campaignId, date, source]`
4. Update rawApiPulls record: `normalized = true`, `normalizedAt = now()`, `schemaVersion = '1.0'`
5. Return count of rows upserted

**Export `processGoogleAdsSync(params: { tenantId, integrationId, dateRange })`:**
1. Get connector config (decrypt tokens from integrations table)
2. Sync campaign hierarchy if needed (upsert into campaigns table)
3. Fetch metrics for date range
4. Store raw pull, normalize, update ingestion coverage
5. Return summary: { recordsIngested, datesProcessed }

Validate incoming GAQL results with Zod before writing to raw_api_pulls.
Use `withTenant(tenantId, ...)` for all database operations.

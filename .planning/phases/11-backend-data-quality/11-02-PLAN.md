    **Python pooled endpoint** (packages/analysis/routers/incrementality.py):

    The `/incrementality/pooled` endpoint currently returns a flat `list` of campaign results. Modify it to return a structured response that includes both the pooled result AND a direct raw score for the target campaign. Import `PooledIncrementalityResponse` from `schemas.responses` and set it as the endpoint's `response_model` so FastAPI validates and documents the return shape.

    1. In `pooled_incrementality_endpoint`, after calling `hierarchical_pooled_estimate(campaigns, cluster_key)`, identify the target campaign. The request dict should include a `target_campaign_id` field (added by the TypeScript caller).

    2. For the target campaign, call `compute_raw_incrementality` directly on that campaign's metrics:
       ```python
       target_campaign_id = request.get("target_campaign_id")
       pooled_results = hierarchical_pooled_estimate(campaigns, cluster_key)

       # Find target campaign's pooled result (adjusted)
       target_result = next((r for r in pooled_results if r["campaign_id"] == target_campaign_id), None)

       if target_result and target_campaign_id:
           # Compute raw score directly for the target campaign
           target_camp = next((c for c in campaigns if c["campaign_id"] == target_campaign_id), None)
           if target_camp:
               from datetime import date as date_type
               target_df = pd.DataFrame(target_camp["metrics"])
               target_df["date"] = pd.to_datetime(target_df["date"]).dt.date
               target_intervention = date_type.fromisoformat(target_camp["intervention_date"])
               try:
                   raw_dict = compute_raw_incrementality(target_df, target_intervention)
               except Exception:
                   # Fallback: use pooled values with reduced confidence
                   raw_dict = {
                       "lift_mean": target_result["lift_mean"],
                       "lift_lower": target_result["lift_lower"],
                       "lift_upper": target_result["lift_upper"],
                       "confidence": target_result["confidence"] * 0.9,
                   }

               return {
                   "adjusted": target_result,
                   "raw": {
                       "campaign_id": target_campaign_id,
                       "status": "pooled_estimate",
                       **raw_dict,
                   },
                   "all_results": pooled_results,
               }

       # Fallback: return legacy flat list format if no target specified
       return pooled_results
       ```

    3. Handle the `< 2 campaigns` fallback path similarly — when falling back to individual estimation, also compute raw for the target and return the same `{adjusted, raw, all_results}` shape.

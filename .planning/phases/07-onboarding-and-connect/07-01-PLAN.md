    **1. Fix GA4EventSelector response shape bug (Pitfall 1 from RESEARCH.md):**
    In `apps/web/components/onboarding/GA4EventSelector.tsx`:
    - Change the `.then()` type annotation from `(data: KeyEvent[])` to `(data: { events: KeyEvent[] })`
    - Change `setEvents(data)` to `setEvents(data.events)`
    - Change `setSelected(new Set(data.map((e) => e.eventName)))` to `setSelected(new Set(data.events.map((e) => e.eventName)))`
    - The POST handler (`handleSave`) is already correct — it sends `{ integrationId, propertyId, selectedEventNames }` in the body, which matches what the API expects. No change needed for POST.

    **2. Fix MarketConfirmationStep session auth:**
    The component currently passes `tenantId` as a prop and includes it in fetch URLs and PUT body. Phase 6 updated all API routes to use session-based auth. Update the component:
    - Remove `tenantId` from Props interface entirely
    - Change `fetch(\`/api/markets?tenantId=${tenantId}\`)` to `fetch('/api/markets')` — session cookie handles auth
    - Remove `tenantId` from all PUT request body objects (the API ignores it per Phase 6 decision, but the component should not send it)
    - Update the component signature to not accept tenantId
    - Also update OutcomeModeSelector.tsx: remove `tenantId` from Props and PUT body (same session auth pattern)

    **3. Modify all 4 OAuth callback routes for popup close pattern (Pitfall 2 from RESEARCH.md):**
    For each of the 4 callback routes (`meta/callback`, `google/callback`, `shopify/callback`, `ga4/callback`):
    - Find the final success `NextResponse.json(...)` return statement
    - Replace it with an HTML response that calls `window.opener.postMessage()` and `window.close()`
    - Keep all error responses as `NextResponse.json(...)` (acceptable in popup — user sees error)
    - The HTML response pattern:
      ```typescript
      const successHtml = `<!DOCTYPE html>
      <html>
      <head><title>Connected</title></head>
      <body>
      <script>
        (function() {
          var data = ${JSON.stringify({
            type: 'oauth_complete',
            platform: '<platform>',       // 'meta', 'google', 'shopify', or 'ga4'
            integrationId: integration.id,
            success: true,
          })};
          if (window.opener && !window.opener.closed) {
            window.opener.postMessage(data, window.location.origin);
          }
          window.close();
        })();
      </script>
      <p>Connected successfully. You can close this window.</p>
      </body>
      </html>`;

      return new NextResponse(successHtml, {
        headers: { 'Content-Type': 'text/html; charset=utf-8' },
      });
      ```
    - For each callback, the variable holding the integration result may differ — read the existing code to find the correct variable name. The meta callback uses `integration` from `saveIntegration()`. Read each callback to find its equivalent.
    - IMPORTANT: Do NOT change anything else in the callbacks — the token exchange, credential storage, backfill enqueue, and nightly sync registration must remain exactly as they are.

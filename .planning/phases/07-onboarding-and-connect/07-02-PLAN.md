    **3. Create BatchMarketConfirmation wrapper (Step 3 — CONTEXT.md: "Batch save — all edits saved when user clicks Next (single API call), can undo before committing"):**
    Create `apps/web/components/onboarding/BatchMarketConfirmation.tsx` — a 'use client' component.
    This wrapper replaces direct usage of MarketConfirmationStep in the wizard. It holds ALL market edits in local React state and only calls the API when the wizard triggers save().

    **State shape:**
    ```typescript
    interface LocalMarket {
      id: string;          // real DB id for existing markets, temp id (crypto.randomUUID()) for new markets
      countryCode: string;
      displayName: string;
      campaignCount: number;
      isConfirmed: boolean;
      isNew?: boolean;     // true for locally-added markets (no DB id yet)
    }

    interface PendingAction {
      action: 'confirm' | 'rename' | 'merge' | 'add' | 'delete';
      id?: string;
      displayName?: string;
      countryCode?: string;
      targetId?: string;
    }
    ```

    **Initialization:**
    - On mount, fetch `GET /api/markets` to load the initial market list into `localMarkets: LocalMarket[]` state
    - Initialize `pendingActions: PendingAction[]` as empty array

    **User interactions (all local state only — NO fetch calls):**
    - **Confirm:** Update `localMarkets` to set `isConfirmed: true`, push `{ action: 'confirm', id }` to pendingActions
    - **Rename (inline edit, onBlur):** Update `localMarkets` displayName, push `{ action: 'rename', id, displayName }` to pendingActions
    - **Delete (X button):** Remove from `localMarkets`, push `{ action: 'delete', id }` to pendingActions. User can undo — add an "Undo" toast or keep a separate undo stack that removes the delete action and restores the market row
    - **Add:** Append to `localMarkets` with a temp UUID and `isNew: true`, push `{ action: 'add', countryCode, displayName }` to pendingActions
    - **Merge (checkbox multi-select + Merge button):**
      - Add checkbox to each market row in the UI
      - When 2+ checkboxes selected, show "Merge" button
      - On Merge click: show shadcn Dialog prompting for merged market name
      - On confirm: keep first selected market as target (rename it to new name), push `{ action: 'rename', id: targetId, displayName: newName }` for the target, and for each other selected market push `{ action: 'merge', id: sourceId, targetId }`. Update `localMarkets` accordingly (merge campaign counts to target, remove sources).
      - The PUT /api/markets API already supports the 'merge' action (confirmed in route.ts).

    **Empty state:** If `localMarkets` is empty after fetch, show "No markets detected yet. Add your markets manually." with Add button (per CONTEXT.md).

    **save() method (called by wizard on Next click):**
    - Exposed via `React.useImperativeHandle` (component wrapped in `React.forwardRef`)
    - If `pendingActions.length === 0`, return immediately (no changes to flush)
    - Call `PUT /api/markets` with body `{ markets: pendingActions }` — single API call with ALL accumulated actions
    - On success: clear `pendingActions`, update `localMarkets` from API response
    - On error: show error toast, do NOT clear pendingActions (user can retry)
    - Return a Promise so the wizard can `await` before advancing

    **canProceed:** Exposed as a getter — true when `localMarkets.length > 0` (at least one market exists in local state)

    **UI:** Reuse the same visual structure as the existing MarketConfirmationStep (Card with rows, inline Input for rename, Badge for campaign count, confirm/delete buttons, add row at bottom) but add checkboxes for merge and wire all handlers to local state instead of fetch calls.

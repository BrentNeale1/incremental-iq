  <action>
1. Create `apps/web/auth.ts` — the Better Auth server instance. Follow Pattern 1 from RESEARCH.md exactly:
   - Import `betterAuth` from `better-auth`
   - Import `drizzleAdapter` from `better-auth/adapters/drizzle`
   - Import `db` from `@incremental-iq/db`
   - Import auth schema tables from `@incremental-iq/db/schema` (authUser, authSession, authAccount, authVerification)
   - Configure: `secret: process.env.BETTER_AUTH_SECRET`, `baseURL: process.env.BETTER_AUTH_URL`
   - Database: `drizzleAdapter(db, { provider: "pg", schema: { user: authUser, session: authSession, account: authAccount, verification: authVerification } })`
   - emailAndPassword: `{ enabled: true, minPasswordLength: 8, maxPasswordLength: 128, sendResetPassword: async ({ user, url }) => { void sendPasswordResetEmail(user.email, url); }, resetPasswordTokenExpiresIn: 3600 }`
   - Session: `{ expiresIn: 60 * 60 * 24 * 30, updateAge: 60 * 60 * 24 }` (30-day with 1-day sliding window — locked decisions)
   - User additionalFields: `{ tenantId: { type: "string", required: true, input: false } }`
   - Do NOT enable `cookieCache` (Pitfall 1 — immediate logout invalidation required by AUTH-03)
   - Create a `sendPasswordResetEmail` helper that uses the existing Resend SDK. Use `void` (fire and forget, not await) to prevent timing attacks.

2. Create `apps/web/auth-client.ts` — the client auth instance. Follow Pattern 2:
   ```typescript
   import { createAuthClient } from "better-auth/react";
   export const authClient = createAuthClient({
     baseURL: process.env.NEXT_PUBLIC_APP_URL,
   });
   ```

3. Create `apps/web/app/api/auth/[...all]/route.ts` — the catch-all API handler. Follow Pattern 3:
   ```typescript
   import { auth } from "@/auth";
   import { toNextJsHandler } from "better-auth/next-js";
   export const { GET, POST } = toNextJsHandler(auth);
   ```

4. Create `apps/web/middleware.ts` — route protection. Follow Pattern 4 from RESEARCH.md:
   - Import `getSessionCookie` from `better-auth/cookies`
   - Auth routes: /login, /signup, /forgot-password, /reset-password
   - No cookie + not auth route → redirect to /login (locked decision: unauthenticated visitors go to /login immediately)
   - Has cookie + auth route → redirect to / (dashboard home)
   - Matcher: `["/((?!api/auth|_next/static|_next/image|favicon.ico).*)"]`
   - SECURITY NOTE in comments: getSessionCookie is optimistic only (cookie existence, not DB validation). Real validation happens in dashboard layout and API routes via auth.api.getSession().

5. Modify `apps/web/app/(dashboard)/layout.tsx`:
   - Convert from 'use client' to a server component wrapper + client inner component pattern
   - The outer server component calls `auth.api.getSession({ headers: await headers() })` — if no session, `redirect("/login")`
   - Extract `session.user.tenantId` and pass it down to the client layout component as a prop
   - The client inner component keeps the existing SidebarProvider, AppSidebar, AppHeader, StaleDataBanner, ExportProvider, and Zustand rehydration logic
   - Replace all `PLACEHOLDER_TENANT_ID` with the real `tenantId` from session
   - The tenantId will be passed as a prop through the component tree (or via a React context if needed for deeper descendants)

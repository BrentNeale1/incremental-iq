## Cross-Phase Integration Issues

### Issue 1: useMarkets hook never called (CRITICAL — MRKT-04)

**From:** Phase 5 (`apps/web/hooks/useMarkets.ts`)
**To:** Dashboard layout (Phase 4/8)
**Impact:** `useDashboardStore.markets` is always `[]`. MarketSelector hidden (`markets.length <= 1`). User cannot select a market. All client-side market filtering is inert.
**Secondary:** `useMarkets` uses `onSuccess` callback (removed in TanStack Query v5) — even if called, Zustand sync would silently fail.
**Fix:** Call `useMarkets()` in `DashboardLayoutClient`. Replace `onSuccess` with `useEffect` on `data` to call `setMarkets(data)`.

### Issue 2: OnboardingWizard resume response mismatch (HIGH — INTG-04 partial)

**From:** Phase 7 (`OnboardingWizard.tsx` line 91)
**To:** Phase 2 (`/api/integrations/status`)
**Impact:** Wizard casts response as `Array<{platform, integrationId}>` but API returns `{globalStatus, integrations: [...], warnings}`. `connectedIntegrations` map is always empty on resume. Wizard always restarts at Step 1.
**Fix:** Read `.integrations` from response and map `{id → integrationId}`.

### Issue 3: Market detection never triggered in onboarding (HIGH — MRKT-01, MRKT-02)

**From:** Phase 7 (`IntegrationConnectStep` / `OnboardingWizard`)
**To:** Phase 5 (`/api/markets/detect`)
**Impact:** After connecting paid channels in Step 1, no market detection runs. Step 3 loads empty markets list. User cannot confirm auto-detected markets.
**Fix:** Call `POST /api/markets/detect` after paid channel connects in `handleIntegrationConnected`.

### Issue 4: Drizzle builder bug in 4 dashboard routes (HIGH — MRKT-03)

**From:** Phase 5 (market filtering)
**To:** Phase 4 (dashboard API routes)
**Routes:** `kpis/route.ts`, `campaigns/route.ts`, `seasonality/route.ts`, `saturation/route.ts`
**Impact:** `query.innerJoin(...)` return value discarded — market filter is silently no-op. Same bug correctly fixed in `engine.ts` by Phase 08-01.
**Fix:** `const filteredQuery = marketId ? query.innerJoin(...) : query` then `filteredQuery.where(...)`.

### Integration Issues (Non-Blocking)

**1. Insights page ignores market filter** (Low)
- `apps/web/app/(dashboard)/insights/page.tsx` calls `useIncrementality(undefined, 'adjusted')` without passing `selectedMarket`
- When market is selected, KPIs/campaigns/recommendations filter but insights scores show all-market data
- Affects MRKT-04 completeness on one of six dashboard pages
- **Fix:** Pass `selectedMarket` to `useIncrementality` and add `marketId` param support

**2. Export data garbled on 2 pages** (Low)
- Health page passes nested `IntegrationSyncHistory[]` objects to SheetJS — `[object Object]` in CSV
- Seasonality page passes nested event objects — same issue
- Executive Overview and Performance pages export correctly (flat `CampaignRow[]` arrays)
- **Fix:** Flatten nested objects before calling `setExportData`

**3. ARCH-03 gate not explicit on recommendations API** (Info)
- `/api/recommendations` does not check `tenants.analysisUnlocked` before generating
- In practice, the scoring pipeline gates on this flag — no scores exist until gate passes
- Risk is minimal but defense-in-depth is missing at the API layer

**4. Budget-change ITS intervention date heuristic** (Low)
- `_getInterventionDate` in `worker.ts` uses campaign start date as ITS intervention for all trigger types
- For `budget_change` triggered jobs, the correct intervention date should be the actual change date
- Pre/post analysis runs but with less precise boundary — acknowledged in code comments
